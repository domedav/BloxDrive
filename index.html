What is BloxDrive?

Instead of paying for Google Drive or Dropbox, BloxDrive stores your files free forever — using Roblox as a backend.

It splits your files into small encrypted chunks, hides them inside Roblox image decals using steganography, and uploads them silently to Roblox's servers. To you, it looks and works like a normal folder on your Linux desktop — just drag, drop, and done.

Nobody — not even Roblox — can read your files. Everything is encrypted on your computer before it ever touches the internet.

Requirements and benefits

🖥️ PC Requirements

  • 🐧Linux — Ubuntu, Debian, Arch, or any modern distro
  • 🐍Python 3.10 or newer
  • 🗄️MySQL or MariaDB — or free TiDB Serverless for multi-device sync
  • 📦FUSE driver: sudo apt install fuse
  • 🎮A free Roblox account with API access

✅ Why BloxDrive?

  • 🆓100% free — no subscriptions, no monthly fees, ever.
  • 🔐Private by design — AES-256-GCM runs locally before upload.
  • 🛡️RAID-5 protection — survive an account ban with zero data loss.
  • 📱Mobile Web UI — stream files to your phone over Wi-Fi.
  • ⚡CDN-speed downloads — served from Roblox's global edge network.

🚀 Up and Running in 2 Steps

Clone the repo first, then two commands and you're done:

git clone https://github.com/domedav/BloxDrive.git && cd BloxDrive

  1. 1

    Install all dependencies

    Run the quick installer — it handles every Python package automatically:

    bash quickinstall.sh
  2. 2

    Launch your drive

    On the very first run, BloxDrive will interactively ask for your Roblox API key and User ID. No config file editing needed.

    ./bloxdrive.sh start

    Your drive is live at /tmp/bloxdrive_mnt — open it in your file manager and drag files in!

🛡️ RAID-5 Redundancy & Recovery

Want to protect your data even if Roblox bans one of your accounts? BloxDrive supports RAID-5 multi-account redundancy — files are striped across accounts with XOR parity, so losing one account loses nothing.

➕ Add an Account

Link another Roblox account to your pool:

./bloxdrive.sh raid add

You can also add accounts through the Web UI RAID Settings.

🔄 Protect Existing Files

Migrate files uploaded before RAID was set up:

./bloxdrive.sh raid protect

Scans all files and re-uploads them with full RAID-5 parity.

🆘 Recovering After a Ban

If an account gets banned, BloxDrive reconstructs the missing data from XOR parity and re-uploads it to a healthy account automatically:

./bloxdrive.sh raid recover

Check your pool health anytime: ./bloxdrive.sh raid status

✨ Full Feature Overview

  • ♾️Effectively limitless storage — Roblox has per-asset size limits, but BloxDrive splits your files into small chunks that each fit within those limits, giving you practically unlimited space.
  • 🔒AES-256-GCM encryption — only you hold the key.
  • 📁Native FUSE filesystem — works with every Linux app, no special client needed.
  • 📲Mobile Web UI — built-in async web file manager for phone access.
  • 🔁Convergent deduplication — identical files are stored once, saving bandwidth.
  • 🌐Multi-device sync — use TiDB Serverless (free) to share files across computers.
  • 🛡️RAID-5 redundancy — survive a single account ban with zero data loss.
  • 🕵️LSB steganography — each encrypted chunk is embedded into the lowest 2 bits of a randomised gradient PNG, hiding binary data inside an innocent-looking image that bypasses moderation AI.
  • 🎲Random asset names — every upload is given a random 8–12 word Faker-generated name, so your real file names are never exposed in your Roblox inventory.
